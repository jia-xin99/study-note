# 同步模式之保护性暂停

## 定义

- 当线程在访问某个对象时，发现条件不满足时，就暂时挂起，等待条件满足时再次访问。用在一个线程等待另一个线程的执行结果。

![](/juc/9.png)

- 有一个结果从一个线程传递到另一个线程，让他们关联同一个GuardedObject；
- 如果有结果不断从一个线程到另一个线程，则使用消息队列（生产者/消费者）；
- **JDK中，join、Future的实现使用此模式；**
- 因为要等待另一方的结果才继续执行，归类到同步模式。

## 实现

```java
class GuardedObject {
    // 结果
    private Object response;
    // 获取结果
    public Object get(){
        synchronized (this) {
            // 没有结果
            while (response == null) {
                // 挂起
                try {
                    this.wait();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            // 如果有结果
            return response;
        }
    }

    // 产生结果
    public void send(Object response) {
        synchronized (this) {
            this.response = response;
            // 通知等待线程
            this.notify();
        }
    }
}
```

## 应用

```java
public static void main(String[] args){
    GuardedObject guardedObject = new GuardedObject();
    // 线程1等待线程2的结果
    new Thread(() -> {
        Object o = guardedObject.get();
        System.out.println(Thread.currentThread().getName() + "获取结果成功，结果是：" + (String)o);
    },"t1").start();

    new Thread(() -> {
        System.out.println(Thread.currentThread().getName()+ "发送结果");
        try {
            Thread.sleep(2323);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        String response = "123456";
        guardedObject.send(response);
    },"t2").start();
}
```

## 超时版GuardedObject

- 超时失效。

```java
class GuardedObjectV2 {
    private Object response;
    private Object lock = new Object();

    // 获取结果（超时处理）
    // timeout：超时时间，表示最多等待多次时间
    public Object get(long timeout) {
        synchronized (lock) {
            // 开始的时间
            long startTime = System.currentTimeMillis();
            // 经过的时间
            long pastTime = 0;
            // 没有结果
            while (response == null) {
                // 更新唤醒的时间
                long waitTime  = timeout - pastTime;
                // 经过的时间超过最大等待时间，直接退出循环
                if(waitTime <= 0) {
                        break;
                }
                // 挂起
                try {
                    lock.wait(waitTime);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                // 唤醒时经过的时间
                pastTime = System.currentTimeMillis()- startTime;
            }
            // 如果有结果
            return response;
        }
    }

    // 产生结果
    public void send(Object response) {
        synchronized (lock) {
            this.response = response;
            // 通知等待线程
            lock.notify();
        }
    }
}
```

## 多任务版GuardedObject

- n个线程等待另外n个对应线程的结果，每个线程对中使用一个GuardedObject对象，因此可以把这多个GuardedObject对象整合成一个新的集合中，可支持多个任务的管理。【当前是1对1发/收消息】

![](/juc/10.png)

### 【1】增加标识id的超时版GuardedObject

```java
class GuardedObject {
    private Object response;
    // 标识Guarded Object
    private int id;
    private Object lock = new Object();

    public GuardedObject(int id) {
        this.id = id;
    }

    public int getId() {
        return this.id;
    }
    public Object get(long timeout) {
        synchronized (lock) {
            long startTime = System.currentTimeMillis();
            long pastTime = 0;
            while (response == null) {
                long waitTime  = timeout - pastTime;
                if(waitTime <= 0) {
                        break;
                }
                try {
                    lock.wait(waitTime);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                pastTime = System.currentTimeMillis()- startTime;
            }
            // 如果有结果
            return response;
        }
    }

    // 产生结果
    public void send(Object response) {
        synchronized (lock) {
            this.response = response;
            // 通知等待线程
            lock.notify();
        }
    }
}
```

### 【2】GuardedObject的聚合类

```java
class MailBoxes {
    // 线程安全：key-value --- id-go对象
    private static Map<Integer,GuardedObject> boxes = new Hashtable<>();
    private static int id = 1;
    // 产生唯一id
    public static synchronized int generateId(){
        return id++;
    }
    public static GuardedObject createGuardedObject() {
        GuardedObject go = new GuardedObject(generateId());
        boxes.put(go.getId(),go);
        return go;
    }

    public static Set<Integer> getIds() {
        return boxes.keySet();
    }

    public static GuardedObject getGuardedObject(int id) {
        // 是只用一次，若要重复使用也可不删
        return boxes.remove(id);
    }
}
```

### 【3】等待消息的线程

```java
class People extends Thread {
    @Override
    public void run() {
        GuardedObject go = MailBoxes.createGuardedObject();
        System.out.println("开始收消息：" + go.getId());
        Object message = go.get(10000);
        if (message == null) {
            System.out.println("接收消息失败");
        }else {
            System.out.println("成功接收到" + go.getId() + "的消息：" + (String)message);
        }
    }
}
```

### 【4】发送消息的线程

```java
class Postman extends Thread {
    private int id;
    private String mail;
    public Postman(int id,String mail) {
        this.id = id;
        this.mail = mail;
    }
    @Override
    public void run() {
        // 发送消息（一对一送信）
        GuardedObject go = MailBoxes.getGuardedObject(id);
        System.out.println(id + "开始送信");
        go.send(mail);
    }
}
```
